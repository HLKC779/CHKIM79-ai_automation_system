Issue #1: Functions Not Connected to Buttons

Symptoms: Clicking buttons does nothing, no errors in console

Common Causes & Solutions:

1. Using onclick attribute (inline handlers)

<!-- CORRECT: Function defined in global scope -->
<button onclick="myFunction()">Click Me</button>

<script>
function myFunction() {
    console.log("Button clicked!");
}
</script>

2. Function not in global scope

<!-- WRONG: Function inside jQuery ready -->
$(document).ready(function() {
    function myFunction() { // Not accessible to onclick
        console.log("Won't work!");
    }
});

<!-- CORRECT: Define globally -->
function myFunction() {
    console.log("Works!");
}

$(document).ready(function() {
    // Other initialization code
});

3. Script loading order issues

<!-- WRONG: Script before HTML -->
<script>
    document.getElementById('myButton').onclick = function() {
        // Element doesn't exist yet!
    };
</script>
<button id="myButton">Click</button>

<!-- CORRECT: Script after HTML or use DOMContentLoaded -->
<button id="myButton">Click</button>
<script>
    document.addEventListener('DOMContentLoaded', function() {
        document.getElementById('myButton').onclick = function() {
            console.log("Works!");
        };
    });
</script>

Issue #2: Event Listeners Not Working

Symptoms: addEventListener doesn't trigger, jQuery click() not working

Solutions:

1. Element doesn't exist when code runs

// WRONG: Element might not exist yet
document.getElementById('myBtn').addEventListener('click', function() {
    console.log('Clicked');
});

// CORRECT: Wait for DOM
document.addEventListener('DOMContentLoaded', function() {
    document.getElementById('myBtn').addEventListener('click', function() {
        console.log('Clicked');
    });
});

2. jQuery not loaded or conflicts

// Check if jQuery is loaded
if (typeof jQuery === 'undefined') {
    console.error('jQuery is not loaded!');
}

// Use jQuery in no-conflict mode
jQuery(document).ready(function($) {
    $('#myButton').click(function() {
        console.log('jQuery click works!');
    });
});

Issue #3: CodePen Specific Issues

Symptoms: Code works locally but not in CodePen

CodePen Settings to Check:

  1. JavaScript Settings: Add external scripts (jQuery, etc.) in Settings → JS
  2. Preprocessors: Make sure you're not using Babel/TypeScript if writing vanilla JS
  3. Console: Open CodePen console to see errors (bottom-left icon)
  4. HTML Comments: Unclosed comments can break everything

Working CodePen Template:

// JS Section (no need for <script> tags)
// Functions are automatically in global scope

function handleClick() {
    console.log("Button clicked!");
    document.getElementById('output').innerHTML = "Working!";
}

// jQuery example (if jQuery is added in settings)
$(document).ready(function() {
    $('#jqueryBtn').click(function() {
        $('#output').append('<br>jQuery works too!');
    });
});

Debugging Checklist

When buttons don't work, check:

  1. ✓ Open browser console (F12) - Any errors?
  2. ✓ Is the function defined? Type its name in console
  3. ✓ Is the function in global scope?
  4. ✓ Are all HTML tags properly closed?
  5. ✓ Is JavaScript enabled in the browser?
  6. ✓ For CodePen: Check JS settings and external scripts
  7. ✓ Try a simple alert() to test if ANY JavaScript works
  8. ✓ Check for typos in function names and IDs
